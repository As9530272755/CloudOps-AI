## 1. Web Terminal 沙盒全面修复

### 1.1 文件复制替代 bind mount
- **问题**: `bind mount` 目标文件必须预先存在，否则 `syscall.Mount` 失败
- **修复**: kubectl 改用 `io.Copy` 复制到沙盒，目标不存在时静默创建
- **文件**: `internal/api/handlers/terminal.go`

### 1.2 `~/.kube/config` 自动创建
- **问题**: kubectl 默认读取 `~/.kube/config`，但沙盒中该路径不存在
- **修复**: 启动沙盒时同时写入 `~/.kube/config`，并创建 `.kube` 目录

### 1.3 `/proc` 挂载 + `/dev/fd` 符号链接
- **问题**: bash 进程替换 `<(...)` 需要 `/proc/self/fd`
- **修复**: 
  - `syscall.Mount("proc", ..., "proc", ...)` 挂载 procfs
  - `os.Symlink("/proc/self/fd", ...)` 创建 `/dev/fd`

### 1.4 代理环境变量过滤
- **问题**: bash login shell 加载宿主机 `/etc/profile.d/proxy.sh`，`http_proxy/https_proxy/no_proxy` 被继承到 chroot 中，导致沙盒内网络连接失败
- **修复**: 启动 bash 前过滤掉所有代理相关环境变量，在 `.bashrc/.bash_profile` 中显式 `unset`

### 1.5 `/dev/urandom` bind mount 占位文件
- **问题**: bind mount 设备文件时目标不存在会报错
- **修复**: 挂载前先用 `os.WriteFile(dstPath, []byte{}, 0644)` 创建占位文件

### 1.6 Shell 体验优化
- **修复**: `.bashrc` / `.bash_profile` 中添加：
  - `alias ls='ls --color=auto'`
  - `alias grep='grep --color=auto'`

### 1.7 集群健康状态检查
- **修复**: WebSocket 连接建立前检查 `ClusterMetadata.HealthStatus`，非 `healthy` 时返回 `403 Forbidden`
- **文件**: `internal/api/handlers/terminal.go`

### 1.8 删除集群时清理终端家目录
- **修复**: `ClusterService.DeleteCluster` 中删除集群后，清理 `/tmp/cloudops-home/cluster-{id}`
- **文件**: `internal/service/cluster_service.go`

### 1.9 Namespace 调整
- **变更**: 沙盒进程命名空间从 `CLONE_NEWPID | CLONE_NEWUSER | CLONE_NEWIPC | CLONE_NEWNS` 调整为 `CLONE_NEWPID | CLONE_NEWUSER | CLONE_NEWIPC`（移除 `CLONE_NEWNS`，避免 mount namespace 冲突）

## 5. 空 Assistant 消息过滤

### 5.1 问题
- 前端在流式请求失败时，本地 state 中残留 `content: ''` 的 assistant 占位消息
- 用户继续在同一会话中发送消息时，前端将空 assistant 带入请求数组
- 后端未过滤直接转发给 Hermes → Kimi API，导致 `400 - the message at position 2 with role 'assistant' must not be empty`

### 5.2 修复
- **后端**: `truncateMessages()` 中增加过滤逻辑，跳过 `role == "assistant" && content == ""` 的消息
- **文件**: `internal/service/ai_service.go`

### 5.3 数据库清理
- 执行 SQL 清理历史残留的空 assistant 消息记录
